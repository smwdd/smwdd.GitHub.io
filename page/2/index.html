第五章
5.1.1 程序和指令的关系
1、计算机中的指令有微指令，机器指令和伪（宏）指令之分
1.机器指令处于硬件和软件的交界面（相当于一个菜谱指定的一个完整做菜过程）
2.微指令是微程序级指令，属于硬件范畴（相当于洗、切、炒等做菜的“微过程”）
3.伪指令是由若干机器组成的指令序列，属于软件范畴（相当于多个菜谱合成一个“大菜”的过程）
4.汇编指令是机器指令的汇编表示形式即符号表示，机器指令和汇编指令一一对应，他们都与具体机器结构有关，都属于机器级指令。机器指令是一个0/1序列，由若干字段组成
5.1.2 目标代码和ISA
1、GCC使用举例
目标文件可用”objdump -d text.o”反汇编为汇编语言程序。比如：0: 55 push %ebp。这三个量分别代表“位移量”、“机器指令”、“汇编指令”。并且编译得到的与反汇编得到的汇编指令形式稍有差异。
2、两种目标文件：text.o：可重定位文件、test：可执行目标文件。test.o中代码从地址0开始，test中代码从地址80483d4（虚拟地址）开始
3、ISA在计算机系统中是必不可少的一个抽象层
1.没有它，软件无法使用计算机硬件
2.没有它，一台计算机不能成为“通用计算机”
5.2.2 IA-32的寄存器组织
1、6个条件标志：ZF：零标志。SF：符号标志。OF：溢出标志。CF：进/借位标志。AF：辅助进位标志（BCD码运算时才有意义）。PF：奇偶标志。
2、3个控制标志：DF：方向标志（自动变址方向是增还是减）。IF：中断允许标志（仅对外部可屏蔽中断有用）。TF：陷阱标志（是否是单步跟踪状态）
5.2.3 IA-32的寻址方式
1、寻址方式：如何根据指令所给定信息得到操作数或操作数地址
2、操作数所在位置：1.指令中：立即寻址。2.寄存器中：寄存器寻址。3.存储单元中（属于存储器操作数，按字节编址）：其他寻址方式
3、存储器的寻址方式与微处理器的工作模式有关
两种工作模式：实地址模式和保护模式。
1.实地址模式：基本用不到。
2.保护模式：需要掌握。加电后进入，采用虚拟存储管理，多任务情况下隔离、保护。

第四章

4.1.1 整数乘法运算
1、通常高级语言中两个n位整数相乘得到的结果通常也是一个n位整数，也即结果只取2n位乘积中的低n位。
所以在计算机内部，x^2不一定>0：x带符号不一定。
x为浮点数则一定。
2、如何判断返回的z是正确值，
int z=xy;
return z;
在这两条语句中添加当!x||z/x==y为真时，是正确的
3、xy的高n位可以用来判断溢出，
1.无符号：若高n位全为0，则不溢出，否则溢出。
2.带符号：若高n位全为0或全为1且等于低n位的最高位，则不溢出。
4、堆溢出漏洞
攻击者可构造特殊参数来触发整数溢出，以一段预设信息覆盖一个已分配的堆缓冲区，造成远程服务器崩溃或者改变内存数据并执行任意代码。
4.2.1 整数除法运算
1、带符号：n位整数除法以n位整数，除-2^(n-1)/-1=2^(n-1)外，其余情况都不会发生溢出。
2、整除0的结果无法用一个机器数表示。
3、不能整除时，采用向0舍入，即截断。
无符号：移出的低位直接舍弃。
带符号负整数：加偏移量（2^k-1），再右移k位。
如：-14/4=-3->(-14+2^2-1)/4=-3。
4.3.1 浮点加减运算
1、两个规格化浮点数分别位A=Ma2^Ea,B=Mb2Eb
A+-B=(Ma+-Mb2^-(Ea-Eb))2Ea
AB=(MaMb)2^(Ea+Eb)
A/B=(Ma/Mb)2(Ea-Eb)
上述运算结果有可能出现一下几种情况：
阶码上溢：SP最大指数为127。
阶码下溢：SP最小指数为-126。
2、对于一个浮点数除以0
1.#INF00表示正无穷。
-1.#INF00表示负无穷。
重点：浮点数加减法基本要点：
假定Xm，Ym分别是X和Y的尾数，Xe和Ye分别是X和Y的阶码
（1）求阶差：delta e=Ye-Xe（若Ye>Xe，结果的阶码为Ye）
（2）对阶：将Xm右移delta e位，尾数变为Xm2^(Xe-Ye)（保留右移部分附件位）
（3）尾数加减：Xm2(Xe-Ye)+-Ym
（4）规格化：
1.当尾数高位为0，则需左规：尾数左移一次，阶码减1，知道MSB（最高位）为1
每次阶码减1后，要判断阶码是否下溢（比最小可表示的阶码还要小）
2.当尾数最高位有进位，需右规：尾数右移一次，阶码加1，知道MSB为1
每次阶码加1后要判断阶码是否上溢（比最大可表示的阶码还要大）
4.3.2 浮点运算的精度
中间结果须在右边加2个附加位
Guard（保护位）：在符号位右边的位。
Round（舍入位）：在保护位右边的位。

第三章
3.1.1 布尔代数和简要逻辑电路
0和1分别代表逻辑值假和真。
三种基本逻辑运算“与”，“或”，“非”，任何一种逻辑表达式都可以写成这三种基本逻辑运算的逻辑组合。例如，异或（只有两个都是不一样的结果才是1，相同就是0）运算的逻辑表达式为：反A与B或A与反B。
组合逻辑电路：没有存储功能，其输出依赖于当前输入。
时序逻辑电路：具有存储功能，其输出不仅依赖于当前输入，还依赖于存储单元的当前状态。
可以利用基本逻辑门电路构成一些具有特定功能的组合逻辑部件（功能部件），如，译码器、编码器、多路选择器、加法器等。
实现一个功能部件的过程：1、用一个真值表描述功能部件的输入和输出之间的关系。2、根据真值表确定逻辑表达式。3.根据逻辑表达式实现逻辑电路。
3.1.2 无符号数加法器
1、一位加法器。（全加器）
两个加数为A和B，低位进位为Cin，和为F，向高位进位为Cout。化简后，逻辑表达式如下：F=A异或B异或Cin，Cout=A与B或A与Cin或B与Cin。
2、n位加法器。
可由n个全加器实现。
重要认识：加法由逻辑部件实现，而其他算术部件都基于加法器和逻辑运算实现，因此，所有算术运算是基于0和1以及逻辑运算实现的！
3、n位带标志加法器。
n位加法器无法用于两个n位带符号的整数相加，无法判断是否溢出。
程序中经常需要比较大小，通过（在加法器中）做减法得到的标志信息来判断。
3.1.3 整数加减运算器和ALU（算术逻辑部件）。
ALU：进行基本算术运算与逻辑运算。
3.3.1 c语言中的各类运算
1、算术运算：无符号数、带符号数、浮点数的+、-、*、/、%运算等。
2、按位运算：
用途：对位串实现掩码（mask）操作或相应的其他处理（主要用于多媒体或状态/控制信息进行处理）
操作：按位或：”|“、按位与：”&“、按位取反：”~“、按位异或：”^“。
3、移位运算：
用途：提取部分信息、扩大或缩小2、4、8……倍。
操作：左移：x<<k；右移：x>>k。
从运算符无法区分逻辑移位还是算术移位，由x的类型确定。
若x为无符号数：高（低）位移出，低（高）位补0，可能溢出！（若高位移出的是1，则左移时发生溢出）
若x为带符号整数：算术左移，算术右移。左移：高位移出，低位补0。可能溢出！（若移出的位不等于新的符号位，则溢出）右移：低位移出，高位补符，可能发生有效数据丢失。
4、逻辑运算：
用途：”||“表示”OR“运算。
”&&“表示”AND“运算。
”！“表示”NOT“运算。
5、位拓展和位截断运算
用途：类型转换时可能需要数据扩展或截断
操作：没有专门操作运算符，根据类型转换前、后数据长短确定是扩展还是截断。
扩展：短转长。（符号位是1，就扩展16个1补全32位）
截断：长转短。
3.4.1 加减运算生成的标志信息
重要认识1：计算机中所有的运算都基于加法器实现。
重要认识2：加法器不知道所运算的是带符号数还是无符号数。
重要认识3：加法器不判定对错，总是取低n位作为结果，并生成标志信息。
标志：零标志：ZF、溢出标志：OF、进/借位标志CF、符号标志SF称为条件标志。

2.3.6乘以常数
先举个例子：【1011】（数值为11）向左移两位得到【101100】（数值为44）相当于乘了2的2次方
并且由于计算机整数乘法比位移和加法的代价要大得多，许多C语言编译器试图以移位、加法、减法来消除很多整数乘以常数的情况。
再举个例子：假设一个程序包含表达式x14。利用14=2^3+2^2+2^1，编译器会将乘法重写为(x<<3)+(x<<2)+(x<<1)，将一个乘法替换为三个位移和两个加法。
甚至可以等于14=2^4-2^1，就可以将乘法重写为(x<<4)-(x<<1)。此时只需要一个减法和两个位移。
2.3.7除以2的幂
在大多数机器上除法要比乘法更慢，需要30个或者更多时钟周期
所以除以2的幂也可以用移位来实现，只不过我们是用的是右移，而不是左移。无符号和补码分别用逻辑位移和算数位移来达到目的。
对于使用算术右移的补码机器，C表达式为：(x<0?x+(1<<k)-1:x)>>k。
2.3.8关于整数运算的最后思考
计算机执行的整数运算实际上是一种模运算形式。表示数字的有限字长限制了可能的值的取值范围，结果运算可能溢出。我们还看到，补码表示提供了一种既能表示负数也能表示正数的灵活方法，同时使用了与执行无符号算术相同的位级实现，这些运算包括像加法、减法、乘法甚至除法，无论运算数是以无符号形式还是补码形式表示的，都有完全一样或者非常类似的位级行为。
2.4浮点数
2.4.1二进制小数
数字权的定义与十进制小数点的位置相关，例如：12.34表示数字110^1+210^0+310^-1+410^-2。
2.4.2IEEE浮点表示
IEEE浮点标准用V=(-1)^sM*2^E的形式来表示一个数：
1.符号s决定这个数是负数(s=1)还是正数(s=0)，而对于数值0的符号为解释作为特殊情况处理。
2.尾数M是一个二进制小数，，它的范围是12-$，或者是01-$。
3.阶码E的作用是对浮点数加权，这个权重是2的E次方幂（可能是负数）。
将浮点数的位表示划分为三个字段，分别对这些值进行编码：
1.一个单独的符号位s直接编码符号s
2.k位的阶码字段exp=ek-1……e1e0编码阶码E。
3.n位小数字段frac=fn-1……f1f0编码位数M，但是编码出来的值也依赖于阶码字段的值是否等于0。
2.4.4舍入
有四种舍入方式，1.向偶数舍入。2.向零舍入。3.向下舍入。4.向上舍入。
2.4.5浮点运算
无符号乘法和除法不具有数学上的结合性和分配性。
2.4.6C语言中的浮点数
所有的C语言版本提供两种不同的浮点数据类型：float（单精度）和double（双精度）。
1.从int转换成float，数字不会溢出，但是可能被舍入。
2.从int或float转换成double，因为double有更大的范围，也有更高的精度，所以能够保留精确的数值。
3.从double转换成float，因为范围更小一些，所以值可能溢出成正无穷或负无穷。另外，由于精确度较小，它还可能被舍入。
4.从float或double转换成int，值将会向零舍入。
