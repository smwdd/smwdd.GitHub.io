一、程序和指令
1、程序和指令的关系
程序由一条一条的指令组成，指令按照顺序存放在内存连续单元

Posted on 2022-08-15

第十二章
12.1.1 重定位的基本概念
符号解析完成后，可进行重定位工作，分三步。
1.合并相同的节
将集合E中的所有目标模块中相同的节合并成新节，例如：所有.text节合并作为执行文件中的。text节
2.对集合D中的定义符号进行重定位（确定地址）
确定新节中所有定义符号在虚拟空间中的地址，例如：为函数确定首地址，进而确定每条指令的地址，为变量确定首地址
完成这一步后，每条指令和每个变量的地址都可确定
3.对引用符号进行重定位（确定地址）
修改.text节和.data节中对每个符号的引用（地址）
需要用到在.rel_data和.rel_text节中保存的重定位信息
12.1.4 符号重定位举例
hello进程的加载运行过程是怎样的
step1：在shell命令行提示符后输入命令：$./hello[enter]
step2：shell命令行解释器构造argv和envp
step3：调用fork()函数，创建一个子进程，与父进程shell完全相同（只读/共享），包括只读代码段，可读写数据段，堆以及用户栈等
step4：调用execve()函数，在当前进程（新创建的子进程）的上下文中加载并运行hello程序。将hello中的.text节、.data 节、.bss节等内容加载到当前进程的虚拟地址空间（仅修改当前进程上下文中关于存储映像的一些数据结构，不从磁盘拷贝代码、数据等内容）
step5：调用hello程序的main()函数，hello程序开始在一个进程的上下文中运行
12.2.1 可执行文件的加载
1.通过调用execve系统调用函数来调用加载器
2.加载器（loader）根据可执行文件的程序（段）头表中的信息，将可执行文件内的代码和数据从磁盘“拷贝”到存储器中（实际上不会真正拷贝，仅建立一种映射）
3.加载后，将pc（EIP）设定指向Entry point（即符号_start处）、最终执行main函数，以启动程序执行
12.3.1 共享库和动态链接概述
一、动态链接用到一个重要概念：
位置无关代码（Position-Independent Code，PIC）
GCC选项-fpic指示生成代码
二、PIC是指：
共享库代码被加载出来的位置可以是不确定的
即使共享库代码的长度发生变化，也不会影响调用它的程序
三、引入PIC的目的
无需修改程序代码即可将共享库加载到任意地址运行
四、共享库内所有引用情况
（1）模块内的过程调用、跳转，采用PC相对偏倚寻址
（2）模块内数据访问，如访问模块内的全局变量和静态变量
（3）模块外的过程调用、跳转
（4）模块外的数据访问，如外部变量的访问

csapp(11)

Posted on 2022-08-12

第十一章
11.1.1 符号和符合表的基本概念
（1）符号解析
目的：将每个模块中应用的符号与某个目标模块中的定义符号建立关联
1. 每个定义符号在代码或数据段中都被分配了存储空间，将应用符号与定义符号建立关联后，就可在重定位时将引用符号的地址重定为相关联的定义符号的地址。
2.本地的局部符号在本模块内定义并引用，因此，其解析较为简单，只要与本模块内唯一的定义符号关联即可。
3.全局符号（外部定义的，内部定义的）的解析涉及到多个模块，故较为复杂
11.2.1 全局符号的强弱
1.函数名和已初始化的全局变量名是强符号
2.未初始化的全局变量名是弱符号
11.1.3 多重定义全局符号的问题
模块间相互引用容易出错，故应尽量避免使用全局变量
一定要用的话一定要遵循以下规则
1.尽量使用本地变量（static），模块内引用不太会出错
2.全局变量要赋初值，使成为强符号，易查出链接错误
3.外部全局变量使用extern，以示其引用的定义在其他模块
注：多重定义全局变量会造成一些意想不到的错误，而且是默默发生的，编译系统不会警告，并会在程序执行很久后才能表现出来，且远离错误引发处。特别是在一个具有几百个模块的大型软件中，这类错误很难修正

第十章
10.2.2 目标文件的三种视图
（1）可重定位目标文件（.o）
其代码和数据可和其他可重定位文件合并为可执行文件
1.每个.o文件对应.c文件的生成
2.每个.o文件代码个数据地址都从0开始
（2）可执行目标文件（linux默认为a.out，windows中的*.exe）
1.包含的代码和数据可以直接被复制到内存并执行
2.代码和数据地址为虚拟地址空间中的地址
（3）共享的目标文件（linux中的*.so）
1.特殊的可重定位目标文件，能在运行时装入到内存并自动被链接，称为贡献库文件
2Windows中称为Dynamic Link Libraries(DLLs)
10.3.1 可重定位文件概述
（1）.symtab节：存放函数名和全局变量（符号表）信息，不包括局部变量
（2）.ret.data节：.data节的重定位信息，用于对模块使用或定义的全局变量进行重定位的信息
（3）.debug节：调试用符号表
（4）strtab节：包含symtab和debug节中符号及节名
（5）Section header table（节头表）：每个节的节名、偏移和大小

第八章
8.1.1 数组的分配和访问
1.buf是在静态区分配的数组，链接后，buf在可执行目标文件数据段中分配了空间
2.aoto型数组分配在栈中，故数组的首址通过EBP来定位
8.2.1 结构类型的分配和访问
（1）结构体成员在内存的存放和访问
1.分配在栈中的auto结构型变量的首地址由EBP或ESP来定位
2.分配在静态区的结构型变量首地址是一个确定的静态区地址
3.结构型变量x各成员首地址可用“基址加偏移量”的寻址方式
（2）结构体数据的分配和访问
按值传递：1.开销大，因为需要对结构体成员整体从静态区复制到栈中，需要很多条mov或其他指令，从而执行时间更长，并占用更多空间和代码空间
8.3.1 数据的对齐方式
（1）按边界对齐（若一个字为32位）
1.字地址：4的倍数（低两位为0）
2.半字地址：2的倍数（低位为0）
3.字节地址：任意
（2）不按边界对齐
坏处：可能会增加访存次数！
8.4.1 越界访问和缓冲区溢出攻击
（1）数组存储区可看成是一个缓冲区，超越数组存储区范围的写入操作称为缓冲区溢出
例如：对于一个有10个元素的char型数组，其定义发缓冲区有10个字节。若写一个字符串到这个缓冲区，那么只要写入的字符串多于9个字符，就会发生写溢出

6.1.1 常用传送指令
（1） 传送指令
通用数据传送指令
MOV：一般传送，包括movb、movw和movl等
MOVS：符号扩展与传送，如movsbw、movswl等
MOVZ：零扩展传送，如movzwl、movbl等
XCHG：数据交换
PUSH/POP：入栈/出栈，如pushl，pushw，popl，popw等
（2）地址传送指令
LEA：加载有效地址，如leal（%edx,%eax）,%eax”的功能为
R[eax]<-R[edx]+R[eax]，执行前，若R[edx]=i
R[eax]=j，则指令执行后，R[eax]=i+j
（3）输入输出指令
IN和OUT：I/O端口寄存器之间的交换
（4）标志传送指令
PUSHF/POPF：将EFLAG压栈，或将栈顶内容送EFLAG
6.2.1 常用定点运算指令
（1）定点算术运算指令
1.加/减运算（影响标志、不区分无/带符号）

（2）加1/减1运算（影响除CF以外的标志、不区分无/带符号）
INC：加，减包括incb、incw、incl等
DEC：减，包括decb、decw、decl等
（3）取负运算（影响标志、若对0取负，则结果为0且CF清0，否则CF置1）
NEG：取负，包括cmpb、cmpw、cmpl等
（4）乘/除运算（不影响标志，区分无/带符号）
MUL/IMUL：无符号乘/带符号乘
DIV/IDIV：带无符号除/带符号除

6.2.2 加法运算的底层实现举例
1.ALU里包含补码的加/减器
带符号整数加减，无符号整数加减。

7.1.1 过程调用概述
一个c过程的大致结构如下
（1）准备阶段
1.形成帧底：push指令和mov指令
2.生成栈帧（如果需要的话）：sub指令或and指令

2.具体处理逻辑，如果遇到函数调用时

3.在EAX中准备返回参数
（3）结束阶段
1.退栈：leave指令或pop指令
2.取返回地址返回：ret指令

Posted on 2022-07-31

第五章
5.1.1 程序和指令的关系
1、计算机中的指令有微指令，机器指令和伪（宏）指令之分
1.机器指令处于硬件和软件的交界面（相当于一个菜谱指定的一个完整做菜过程）
2.微指令是微程序级指令，属于硬件范畴（相当于洗、切、炒等做菜的“微过程”）
3.伪指令是由若干机器组成的指令序列，属于软件范畴（相当于多个菜谱合成一个“大菜”的过程）
4.汇编指令是机器指令的汇编表示形式即符号表示，机器指令和汇编指令一一对应，他们都与具体机器结构有关，都属于机器级指令。机器指令是一个0/1序列，由若干字段组成
5.1.2 目标代码和ISA
1、GCC使用举例
目标文件可用”objdump -d text.o”反汇编为汇编语言程序。比如：0: 55 push %ebp。这三个量分别代表“位移量”、“机器指令”、“汇编指令”。并且编译得到的与反汇编得到的汇编指令形式稍有差异。
2、两种目标文件：text.o：可重定位文件、test：可执行目标文件。test.o中代码从地址0开始，test中代码从地址80483d4（虚拟地址）开始
3、ISA在计算机系统中是必不可少的一个抽象层
1.没有它，软件无法使用计算机硬件
2.没有它，一台计算机不能成为“通用计算机”
5.2.2 IA-32的寄存器组织
1、6个条件标志：ZF：零标志。SF：符号标志。OF：溢出标志。CF：进/借位标志。AF：辅助进位标志（BCD码运算时才有意义）。PF：奇偶标志。
2、3个控制标志：DF：方向标志（自动变址方向是增还是减）。IF：中断允许标志（仅对外部可屏蔽中断有用）。TF：陷阱标志（是否是单步跟踪状态）
5.2.3 IA-32的寻址方式
1、寻址方式：如何根据指令所给定信息得到操作数或操作数地址
2、操作数所在位置：1.指令中：立即寻址。2.寄存器中：寄存器寻址。3.存储单元中（属于存储器操作数，按字节编址）：其他寻址方式
3、存储器的寻址方式与微处理器的工作模式有关
两种工作模式：实地址模式和保护模式。
1.实地址模式：基本用不到。
2.保护模式：需要掌握。加电后进入，采用虚拟存储管理，多任务情况下隔离、保护。

第四章

4.1.1 整数乘法运算
1、通常高级语言中两个n位整数相乘得到的结果通常也是一个n位整数，也即结果只取2n位乘积中的低n位。
所以在计算机内部，x^2不一定>0：x带符号不一定。
x为浮点数则一定。
2、如何判断返回的z是正确值，
int z=xy;
return z;
在这两条语句中添加当!x||z/x==y为真时，是正确的
3、xy的高n位可以用来判断溢出，
1.无符号：若高n位全为0，则不溢出，否则溢出。
2.带符号：若高n位全为0或全为1且等于低n位的最高位，则不溢出。
4、堆溢出漏洞
攻击者可构造特殊参数来触发整数溢出，以一段预设信息覆盖一个已分配的堆缓冲区，造成远程服务器崩溃或者改变内存数据并执行任意代码。
4.2.1 整数除法运算
1、带符号：n位整数除法以n位整数，除-2^(n-1)/-1=2^(n-1)外，其余情况都不会发生溢出。
2、整除0的结果无法用一个机器数表示。
3、不能整除时，采用向0舍入，即截断。
无符号：移出的低位直接舍弃。
带符号负整数：加偏移量（2^k-1），再右移k位。
如：-14/4=-3->(-14+2^2-1)/4=-3。
4.3.1 浮点加减运算
1、两个规格化浮点数分别位A=Ma2^Ea,B=Mb2Eb
A+-B=(Ma+-Mb2^-(Ea-Eb))2Ea
AB=(MaMb)2^(Ea+Eb)
A/B=(Ma/Mb)2(Ea-Eb)
上述运算结果有可能出现一下几种情况：
阶码上溢：SP最大指数为127。
阶码下溢：SP最小指数为-126。
2、对于一个浮点数除以0
1.#INF00表示正无穷。
-1.#INF00表示负无穷。
重点：浮点数加减法基本要点：
假定Xm，Ym分别是X和Y的尾数，Xe和Ye分别是X和Y的阶码
（1）求阶差：delta e=Ye-Xe（若Ye>Xe，结果的阶码为Ye）
（2）对阶：将Xm右移delta e位，尾数变为Xm2^(Xe-Ye)（保留右移部分附件位）
（3）尾数加减：Xm2(Xe-Ye)+-Ym
（4）规格化：
1.当尾数高位为0，则需左规：尾数左移一次，阶码减1，知道MSB（最高位）为1
每次阶码减1后，要判断阶码是否下溢（比最小可表示的阶码还要小）
2.当尾数最高位有进位，需右规：尾数右移一次，阶码加1，知道MSB为1
每次阶码加1后要判断阶码是否上溢（比最大可表示的阶码还要大）
4.3.2 浮点运算的精度
中间结果须在右边加2个附加位
Guard（保护位）：在符号位右边的位。
Round（舍入位）：在保护位右边的位。
